## 2024-05-03 - [Accessible Form Fields without Visual Changes]
**Learning:** This app frequently relies on visual proximity to indicate input meaning, using minimal visual labels (or relying on grid headers) for items like scoring inputs and custom toggle switches. Using isolated `aria-label`s on inputs drops context and limits the ability to associate helper text natively.
**Action:** Use `.sr-only` explicitly linked `<label>` tags (using `for` and `id`) to maintain the minimalist visual design while correctly identifying input controls to screen readers. For complex custom controls like toggle switches with accompanying descriptions, assign `id`s to the visual label and description text elements and map them directly using `aria-labelledby` and `aria-describedby`.
